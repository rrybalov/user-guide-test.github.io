Search for a client

How to look up whether a client already exists before you do anything else in ANSA.

Tool
ANSA
Last reviewed
2013 — source
Status
Needs review
Maintained by
App Support
Why this matters. Searching is how you avoid a duplicate. It changes nothing in the system, so it’s always safe to run — and running it first is what tells you whether to Show an existing client or move on to registration.

Steps

  1. Open the client search.Sign in through Okta, open the assessment tile, then go to the CANS / ANSA menu and choose Client > Search.
  2. Set how each field should match.For First and Last Name, pick a match type: Sounds Like, Starts With, or Is. “Sounds Like” casts the widest net and is the safest first try.
  3. Enter what you know.Only Last Name is required. Add Date of Birth, Gender, SSN, or MCI number to narrow the results.
  4. Select Search and read the result.You’ll get either a list of matches or a “No Match Found” message — see below.

What your result means

Matches

One or more clients are listed

Open a row to check the Info, Address, and Assessment History tabs and confirm it’s the right person. When you’re sure, select the row and choose Show to bring that client into focus.

No match

“No Match Found”

Select OK, then choose New to register the client. Continue to the Register a new client guide.

If something goes wrong

What to do
No results, but you expect the person to existTry again — loosen the match to Sounds Like and search on Last Name only. Spelling and nicknames are the usual culprits.
Too many results to tell them apartNarrow — add Date of Birth or SSN, then use the Info and Assessment History tabs to confirm.
Opened the wrong client with ShowRecoverable — selecting Show changes nothing. Return to Client > Search and run it again.
