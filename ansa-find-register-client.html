Find & register a client

One continuous task: search for the client first, and register a new one only if the search comes back empty. Both steps live on this page.

Why the order matters. Every client lives in the county’s Master Client Index (MCI). If you register without searching, you can create a second record for someone who already exists — splitting their history. So you always search first; you only register when there’s truly no match.

1

Search for a client

DHS Providers

Searching changes nothing in the system, so it’s always safe to run — and it’s what tells you whether to open an existing client or move on to registration.

Steps

  1. Open the client search.Sign in through Okta, open the assessment tile, then go to the CANS / ANSA menu and choose Client > Search.
  2. Set how each field should match.For First and Last Name, pick a match type: Sounds Like, Starts With, or Is. “Sounds Like” casts the widest net and is the safest first try.
  3. Enter what you know.Only Last Name is required. Add Date of Birth, Gender, SSN, or MCI number to narrow the results.
  4. Select Search and read the result.You’ll get either a list of matches or a “No Match Found” message — see below.

What your result means

Matches

One or more clients are listed

Open a row to check the Info, Address, and Assessment History tabs and confirm it’s the right person. When you’re sure, select the row and choose Show to bring that client into focus. You’re done — no need to register.

No match

“No Match Found”

Select OK, then continue to Step 2 · Register a new client just below.

If something goes wrong

What to do
No results, but you expect the person to existTry again — loosen the match to Sounds Like and search on Last Name only. Spelling and nicknames are the usual culprits.
Too many results to tell them apartNarrow — add Date of Birth or SSN, then use the Info and Assessment History tabs to confirm.
Opened the wrong client with ShowRecoverable — selecting Show changes nothing. Return to Client > Search and run it again.
Only if the search finds no one
2

Register a new client

DHS Providers

Start here only after a search returns “No Match Found.” The New button doesn’t turn on until a search has run — that’s by design.

The Verify Individual step is your safety check: it compares what you typed against everyone already in MCI, so a person who exists under a slightly different spelling still gets caught before a duplicate is created.

Steps

  1. Select New.From the search screen, after “No Match Found,” choose New to open a blank client record.
  2. Enter the core identifiers.First Name, Last Name, SSN, Date of Birth, and Gender.
  3. Select Verify Individual.This checks your entry against the Master Client Index.
  4. Handle the match result.If MCI returns a possible match of 95% or higher, you must select the existing person rather than create a new record. If you’re certain it’s a different person, the Service Desk can override. If there’s no match, continue.
  5. Complete the demographics.Add Race and Ethnicity to finish the person’s profile.
  6. Add an address.Choose New (or Edit), enter the address, and validate against Postal. If it won’t validate but you know it’s correct, use Save Without Verification.
  7. Select Save.The client is registered and ready to assess.

If something goes wrong

What to do
Verify Individual returns a 95%+ match but you believe it’s a different personCheck first — open the match and compare DOB and SSN. If it truly is someone else, contact the Service Desk for an override; don’t force a duplicate.
You registered someone who turned out to already existRecoverable — call the Service Desk (412-350-4357) to merge or correct the records.
The address won’t validate against PostalOption — if you’ve confirmed it’s right, use Save Without Verification rather than altering a correct address.
The New button is greyed outExpected — run a search first. New only enables after a search has run.
